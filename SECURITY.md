# Security Policy

## Reporting a Vulnerability

Report security issues privately to security@anatolykoptev.com with a clear description and reproduction steps. Do not open public issues for security vulnerabilities.

## Disclosure

Vulnerable versions are patched first, then disclosed after a reasonable release window.

## Scope

- Signing secret handling.
- Source URL allowlist and SSRF protection.
- Admin UI sanitization and capability checks.
- Health check endpoint behavior.
- Uninstall cleanup.
